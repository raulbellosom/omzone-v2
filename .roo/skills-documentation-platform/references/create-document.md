# Create Document Skill

Generates documentation from structured data sources including JSON, YAML, API responses, or database schemas.

## Overview

This skill transforms structured data into well-formatted markdown documentation with proper frontmatter, heading hierarchy, and content organization.

## When to Use

- Creating API reference documentation from OpenAPI/Swagger specs
- Generating component documentation from JSON schemas
- Creating guides from structured configuration examples
- Building reference pages from database schema definitions

## Input Formats

### JSON Schema Input
```json
{
  "entity": "User",
  "attributes": [
    { "name": "id", "type": "string", "description": "Unique identifier" },
    { "name": "email", "type": "string", "description": "User email address" },
    { "name": "createdAt", "type": "datetime", "description": "Creation timestamp" }
  ],
  "methods": [
    { "name": "create", "description": "Create a new user" },
    { "name": "update", "description": "Update user details" }
  ]
}
```

### YAML Configuration Input
```yaml
entity: User
attributes:
  - name: id
    type: string
    description: Unique identifier
  - name: email
    type: string
    description: User email address
```

## Output Format

Generated markdown file with:
- YAML frontmatter (title, description, category, order)
- H1 heading matching entity name
- Attribute tables with type and description columns
- Method/endpoint documentation sections
- Code examples where applicable

## Frontmatter Template

```yaml
---
title: {entityTitle}
description: {autoGeneratedDescription}
category: {category}
tags: [{tags}]
order: {order}
version: {version}
author: {author}
createdAt: {ISO8601timestamp}
updatedAt: {ISO8601timestamp}
---
```

## Code Examples

### Basic Entity Documentation

**Input:**
```
Entity: Product
Category: Commerce
Attributes: id (string), name (string), price (number), description (string)
```

**Output:**
```markdown
---
title: Product
description: Represents a product in the commerce system
category: Commerce
tags: [entity, product, catalog]
order: 1
---

# Product

## Attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| id | string | Unique product identifier |
| name | string | Product display name |
| price | number | Product price in default currency |
| description | string | Product description text |

## Related Topics

- [Category](./category.md) - Products belong to categories
- [Inventory](./inventory.md) - Track product stock levels
```

### API Endpoint Documentation

**Input:**
```json
{
  "method": "POST",
  "path": "/api/v1/products",
  "requestBody": {
    "name": { "type": "string", "required": true },
    "price": { "type": "number", "required": true }
  },
  "responses": {
    "201": { "description": "Product created successfully" },
    "400": { "description": "Invalid request data" }
  }
}
```

**Output:**
```markdown
---
title: Create Product
description: Create a new product via API
category: API Reference
tags: [api, products, post, create]
order: 1
---

# Create Product

**Endpoint:** `POST /api/v1/products`

| Field | Type | Required | Description |
|-------|------|----------|-------------|

| price | number | Yes | Product price |

### 400 Bad Request
```

## Validation Rules

1. **Entity names** must be title-cased (Product, UserProfile)
2. **Attribute types** must map to known types (string, number, boolean, array, object, datetime)
3. **Category** must exist in the navigation hierarchy
4. **Order** must be a positive integer
5. **Tags** must be lowercase, hyphen-separated

## Error Handling

| Error | Cause | Resolution |
|-------|-------|------------|
| Missing entity name | Input lacks entity definition | Provide entity field |
| Unknown type | Attribute type not recognized | Use valid type or add to type mapping |
| Invalid category | Category not in hierarchy | Use existing category or create new one |

## Integration Points

- **format-code**: Code examples are automatically passed to format-code for syntax highlighting
- **generate-toc**: Headings are extracted for TOC generation
- **metadata-management**: Author and timestamps added automatically
- **cross-reference**: Related topics links added post-generation
